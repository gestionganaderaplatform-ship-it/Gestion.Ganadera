# GANADERO SaaS · Mapa de matrices transversales

## Propósito
Este paquete consolida las matrices transversales del producto para conectar los procesos ya definidos con la siguiente capa de análisis funcional, modelo lógico y construcción.

## Alcance
Estas matrices cubren fase 1 y fase 2, tomando como base el documento funcional oficial del producto y las especificaciones por proceso ya generadas.

## Matrices incluidas
1. 01_matriz_de_eventos_del_animal.md
2. 02_matriz_de_estados_derivados_del_animal.md
3. 03_matriz_de_permisos_por_rol.md
4. 04_matriz_de_correccion_y_anulacion.md
5. 05_matriz_de_catalogos_y_parametros.md
6. 06_matriz_de_campos_comunes_por_proceso.md
7. 07_matriz_de_impacto_en_historial.md
8. 08_matriz_de_aplicabilidad_por_canal_web_app.md

## Uso recomendado
El orden recomendado de trabajo es:
1. eventos del animal
2. estados derivados
3. permisos
4. corrección y anulación
5. catálogos y parámetros
6. campos comunes
7. historial
8. canal web/app

## Observación
Estas matrices no reemplazan los documentos por proceso. Los complementan y sirven como puente hacia:
- modelo funcional de entidades
- modelo lógico de datos
- reglas de auditoría
- diseño de pantallas
- contratos de servicios
- backlog de construcción

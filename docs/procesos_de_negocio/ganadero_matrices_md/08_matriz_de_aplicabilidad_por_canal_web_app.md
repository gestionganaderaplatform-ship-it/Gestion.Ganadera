# Matriz de aplicabilidad por canal web / app

## Objetivo
Definir qué procesos tienen mejor afinidad funcional con web, app o ambos, siguiendo la estrategia del producto.

| Proceso | Web fase inicial | App futura | Justificación principal |
|---|---|---|---|
| Registro de existente | Sí | Baja | Alta carga inicial, revisión, posible masivo |
| Compra | Sí | Media | Puede ocurrir en campo, pero requiere contexto y validación más amplia |
| Pesaje | Sí | Alta | Alta frecuencia, terreno, captura rápida |
| Movimiento de potrero | Sí | Alta | Muy operativo, repetitivo y de campo |
| Venta | Sí | Media | Sensible, menos frecuente, más control administrativo |
| Muerte | Sí | Media | Operativo, pero requiere control y trazabilidad |
| Traslado entre fincas | Sí | Baja/Media | Más sensible y menos frecuente |
| Vacunación | Sí | Alta | Repetitiva, de terreno, candidata clara a app |
| Tratamiento sanitario | Sí | Alta | Operativo y de campo |
| Palpación o revisión reproductiva | Sí | Media/Alta | Campo especializado |
| Destete | Sí | Media/Alta | Campo, pero con contexto funcional |
| Cambio de categoría | Sí | Baja/Media | Mejor controlado desde web, salvo confirmación móvil futura |
| Descarte | Sí | Media | Sensible, pero puede originarse en campo |
| Nacimiento | Sí | Alta | Muy natural en captura móvil de terreno |

## Regla madre
- La web sale primero como canal completo.
- La app llega después para procesos de alta frecuencia, velocidad y trabajo en terreno.
